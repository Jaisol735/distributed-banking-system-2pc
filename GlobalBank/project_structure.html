📄 Every File Explained

✨ New Feature — Bank Selection Shows Supabase DB Info

When the user opens Open Account and clicks a bank button, selectBank() in index.html calls GET /bank_config/<BANK> on the Flask server. Flask reads Server/supabase.json and returns the host, database, and user for that bank. The password is never sent to the browser. A live DB card appears in the signup form so the user can see which Supabase project their account will be stored in.

FileLayerWhat it does
Server/config.jsonConfigThe one file to edit. Set tailscale_ip to PC-2's Tailscale IP (tailscale ip -4). Read by every server via config_loader.py.
Server/config_loader.pyConfigShared helper imported by all 7 servers. Reads config.json. Builds URL lists: Tailscale IP first, 127.0.0.1 as fallback. Zero hardcoded IPs anywhere else.
Server/supabase.jsonConfigAll 4 banks' Supabase credentials. Read by app.py /bank_config. Password never forwarded to browser.
Server/world_directory.jsonConfigGlobal bank → country → port routing map. Used by country servers to resolve receiver bank location.
world.pyWorld :4000Health-monitors countries every 5 s. Validates cross-country transfers (both country + receiver bank online?). Never touches money.
world_recovery.pyWorld :4000Resets all country statuses to OFF in world_log.json. Removes stuck transaction records. Health monitor re-discovers on startup.
world_log.jsonWorld :4000Persists country ON/OFF status between restarts.
india.py / usa.pyCountryHealth-monitors local banks every 5 s. Routes same-country transfers (ACK only — 2PC is the bank's job). Forwards cross-country requests to World.
india_recovery.py / usa_recovery.pyCountryResets all bank statuses to OFF in registry JSON so health monitor starts fresh.
india_registry.json / usa_registry.jsonCountryLive map of bank name → port + status + fail_count. Written by health monitor every 5 s.
sbi.py / hdfc.py / jpmc.py / boa.pyBankAll transaction logic: bcrypt signup/login, PIN-gated balance, FIFO deposit/withdraw, full 2PC transfer. Exposes /receive_prepare, /receive_commit, /receive_abort for cross-bank coordination.
sbi_recovery.py … boa_recovery.pyBankRefunds PREPARED senders + ABORT. Aborts PREPARED receivers. Resets PROCESSING queue rows → PENDING for FIFO re-processing.
bank_log.json (×4)BankBank name, port, and Supabase connection credentials. Read on every DB connect call. Each bank has its own file.
frontend/app.pyFrontendFlask server on PC-1. Serves index.html. /config → port map. /bank_config/<BANK> → Supabase DB info (no password). /proxy/<port>/<path> → reverse-proxy to PC-2. Zero CORS issues.
frontend/templates/index.htmlFrontendFull banking UI. Login, signup (with live DB info card on bank select), balance, deposit, withdraw, send money with 2PC animation, server log panel.
migration.sqlDBAdds bank column to tx_requests table + performance indexes. Run once per Supabase instance (SBI, HDFC, JPMC, BOA).
🚀 Setup Order
1

Edit Server/config.json — set your Tailscale IP

On PC-2 run tailscale ip -4 to get your Tailscale IP, then paste it into tailscale_ip. This is the only file you ever need to edit to configure networking.

2

Run migration.sql on each Supabase project

Open each of the 4 Supabase projects → SQL Editor → paste and run migration.sql. Do this for SBI, HDFC, JPMC, and BOA separately.

3

Install dependencies on PC-2

pip install flask psycopg2-binary requests bcrypt
4

Start all servers on PC-2 (in this order)

cd Server/world_server  &&  python world.py
cd Server/India         &&  python india.py
cd Server/USA           &&  python usa.py
cd Server/India/SBI     &&  python sbi.py
cd Server/India/HDFC    &&  python hdfc.py
cd Server/USA/JPMC      &&  python jpmc.py
cd Server/USA/BOA       &&  python boa.py
5

Start the frontend on PC-1

pip install flask requests
cd frontend
python app.py --server-ip <PC-2 Tailscale IP>

Then open http://localhost:8080 in your browser.

⚖ System Guarantees
R1
Zero CORS. Browser talks only to localhost:8080. Flask proxies every call to PC-2 via Tailscale. No cross-origin request ever leaves the browser.
R2
One-file IP config. Edit tailscale_ip in Server/config.json once. All 7 servers read it via config_loader.py — nothing else to change.
R3
Password never leaves the server. supabase.json is read only by Flask backend. /bank_config returns host/database/user — the password key is deliberately absent from the response.
R4
2PC atomicity. If receiver PREPARE fails after sender debit → sender is immediately refunded. If receiver COMMIT fails → abort + refund. Money is never lost.
R5
FIFO + deadlock-free. Deposits and withdrawals use FOR UPDATE SKIP LOCKED. Earlier requests always complete first. Workers never block each other.
R6
Crash recovery. Every server runs its *_recovery.py on startup. PREPARED senders are refunded. PREPARED receivers are aborted. PROCESSING queue rows are re-queued to PENDING.
